Waarp R66-TaskOptions

We focus here on the several tasks that are possible to run before a transfer starts (pre action), after a transfer is finished correctly (post action) or after an error occurs (either in pre or post action or during transfer: error action).

 

Those actions are defined in one rule. Each rule contains 2 parts:

  1. Sender actions: A host is a Sender if it is the requester on a SEND rule or if it is the requested on a RECV rule. Those are split in Pre, Post transfer and in Error transfer. 
  2. Receiver actions: A host is a Sender if it is the requester on a RECV rule or if it is the requested on a SEND rule. Those are split in Pre, Post transfer and in Error transfer.

Each action could be on pre, post or error step, each step can have several actions.

 

FORMAT

Each action is of one of the following items:

  • Actions to log or trace
    • LOG: log a message
    • SNMP: send a trap or inform using SNMP
  • Actions to change the current file position or copying it or validating its existence or some properties
    • COPY: copy the file (same name) in other place
    • COPYRENAME: copy the file (new name) in other place
    • MOVE: move the file (same name) in other place
    • MOVERENAME: move the file (new name) in other place
    • LINKRENAME: try to link the file, or copy it if it cannot (new name) in other place
    • RENAME: rename the file (only virtual file, not real one)
    • DELETE: delete current file
    • VALIDFILEPATH: validate the existence of the current file
    • CHKFILE: check the File size according to limit specified or to available space under Working and Receive directories
  • Actions to act on the file
    • TRANSCODE: transcode the file to a new coding
    • TAR: tar a set of files / untar a file to a directory
    • ZIP: zip a set of files / unzip a file to a directory
    • UNZEROED: check if the file size is zero (0), and if so add one blank character to it to allow transfer.
    • CHMOD: change the access mode of the file (Read, Write, Execute) for User or All (Java limitation).
  • Actions for external execution
    • EXEC: execute an external command
    • EXECMOVE: execute an external command and take the result as new current file name
    • EXECOUTPUT: execute an external command, and in error returns the last line string and eventually rename the current file
    • EXECJAVA: execute a Java class command
  • Actions to transfer file
    • TRANSFER: R66 async send generated
    • RESCHEDULE: in case of error, can reschedule the current transfer in the future
    • FTP: FTP sync send generated

Example:

      <tasks>

     <task>

        <type>MOVE</type>

        <path>/pathout/</path>

        <comment>move the file to /pathout/#TRUEFILENAME#</comment>

        <delay>0</delay>

     </task>

     <task>

        <type>EXEC</type>

        <path>#HOMEPATH#/pathexec/monscript #TRUEFULLPATH# #ORIGINALFILENAME# #FILESIZE# #RULE# %s %d #REMOTEHOST#</path>

        <comment>information passed by transfer is "a_string_without_blank 9" and replaced respectively in %s and %d</comment>

        <delay>30000</delay>

        <comment>maximum 30 seconds to execute this script</comment>

        <comment>execute: /r66home/pathexec/monscript /truepath/truefilename originalfilename 9999 rulename a_string_without_blank 9 remotehostid</comment>

     </task>

   </tasks>
